PRD Refinement

Automatically analyze and refine Product Requirements Documents by identifying gaps, suggesting improvements, and adding technical specifications.

How to trigger:

Add the label openhands:refine to any GitHub issue containing a PRD.

What it does:

  • Analyzes the PRD for completeness
  • Identifies ambiguous requirements
  • Suggests acceptance criteria
  • Adds technical considerations

Example:

1. Create an issue with your PRD
2. Add label: openhands:refine
3. OpenHands will comment with improvements

Spec Generator

Generate detailed technical specifications from high-level requirements, including architecture diagrams, API contracts, and implementation plans.

How to trigger:

Add the label openhands:spec to any GitHub issue with feature requirements.

What it does:

  • Creates detailed technical specs
  • Defines API contracts
  • Outlines data models
  • Proposes implementation approach

Example:

1. Create an issue describing the feature
2. Add label: openhands:spec
3. Receive a detailed technical specification

Refactor

Intelligently refactor code in pull requests to improve code quality, readability, and maintainability while preserving functionality.

How to trigger:

Add the label openhands:refactor to any pull request.

What it does:

  • Improves code structure
  • Extracts reusable functions
  • Applies design patterns
  • Enhances naming conventions

Example:

1. Open a PR with code changes
2. Add label: openhands:refactor
3. OpenHands suggests refactoring improvements

Dependency Upgrades

Automatically upgrade project dependencies, handle breaking changes, and ensure compatibility across your codebase.

How to trigger:

Add the label openhands:upgrade to an issue specifying the dependencies to upgrade.

What it does:

  • Updates dependency versions
  • Fixes breaking changes
  • Updates deprecated APIs
  • Runs tests to verify compatibility

Example:

1. Create issue: "Upgrade React to v18"
2. Add label: openhands:upgrade
3. Receive a PR with all necessary changes

Bug Fixer

Automatically investigate and fix bugs based on issue descriptions, error logs, and reproduction steps.

How to trigger:

Add the label bug to any GitHub issue with bug details.

What it does:

  • Analyzes bug reports
  • Identifies root causes
  • Creates a fix PR
  • Adds regression tests

Example:

1. Create issue with bug description
2. Add label: bug
3. OpenHands creates a PR with the fix

PR Review

Perform comprehensive code reviews on pull requests, identifying issues, suggesting improvements, and ensuring code quality standards.

How to trigger:

Add the label openhands:review to any pull request.

What it does:

  • Reviews code for issues
  • Checks for security vulnerabilities
  • Suggests optimizations
  • Verifies test coverage

Example:

1. Open a pull request
2. Add label: openhands:review
3. Receive detailed code review comments

Vulnerability Remediation

Automatically fix security vulnerabilities identified by security scanners and dependency audits.

How to trigger:

Add the label openhands:remediate to security advisory issues.

What it does:

  • Analyzes vulnerability reports
  • Updates affected dependencies
  • Patches vulnerable code
  • Validates the fix

Example:

1. Dependabot creates security alert
2. Add label: openhands:remediate
3. OpenHands creates a fix PR

CI Failure Fixer

Automatically analyze and fix CI pipeline failures, including test failures, linting errors, and build issues.

How to trigger:

Automatic: Triggers whenever a CI workflow fails.

What it does:

  • Analyzes CI failure logs
  • Identifies the root cause
  • Creates a fix commit
  • Re-runs the pipeline

Example:

1. CI pipeline fails on your PR
2. OpenHands automatically analyzes the failure
3. A fix commit is pushed to your branch
